# Implementation Plan: Sequential Profiling & Optimization (Phase 8.1g)

**Branch**: `018-sequential-profiling-optimization` | **Date**: 2026-02-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/018-sequential-profiling-optimization/spec.md`

## Summary

Phase 8.1g instruments the multifrontal factorization loop with per-supernode timing, optimizes the top sequential allocation hotspots, collects performance baselines across the full SuiteSparse suite, and produces a workload analysis report that recommends a parallelism strategy for Phase 8.2. All work builds on existing infrastructure (ProfileSession, MemoryTracker, benchmarking framework) and preserves correctness on all 65 SuiteSparse matrices.

## Technical Context

**Language/Version**: Rust 1.87+ (edition 2024)
**Primary Dependencies**: faer 0.22, metis-sys 0.3.x, serde/serde_json (existing); no new dependencies
**Storage**: JSON files in `target/benchmarks/baselines/` for baseline data; Chrome Trace JSON for profiling
**Testing**: `cargo test` (362+ unit tests) + `cargo test -- --ignored --test-threads=1` (65 SuiteSparse matrices)
**Target Platform**: Linux (RSS tracking via `/proc/self/status`)
**Project Type**: Single Rust library with examples
**Performance Goals**: No regression in factorization time; measurable reduction in allocation count on large fronts
**Constraints**: All optimizations must preserve reconstruction error < 1e-12 and backward error < 5e-11 for all 65 SuiteSparse matrices
**Scale/Scope**: 65 SuiteSparse matrices (dimensions 30-65,000+), per-supernode granularity (10s to 1000s of supernodes per matrix)
