# Implementation Plan: Test Matrix Collection Assembly

**Branch**: `002-test-matrix-collection` | **Date**: 2026-02-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-test-matrix-collection/spec.md`

## Summary

Build a comprehensive test matrix collection for SSIDS solver validation, comprising 70+ sparse symmetric matrices from four sources: hand-constructed matrices with exact factorizations, curated SuiteSparse Collection matrices classified by difficulty, SPRAL-inspired test patterns, and paper-referenced benchmark problems. Matrices stored in Matrix Market format with a JSON metadata index. SuiteSparse matrices downloaded via `ssgetpy` (Python) and tracked with Git LFS. Hand-constructed matrices generated by a Python script with analytically verified LDL^T factorizations.

## Technical Context

**Language/Version**: Python 3.8+ (scripts for matrix generation and download)
**Primary Dependencies**: `ssgetpy` (SuiteSparse query/download), `numpy`/`scipy` (matrix construction and factorization verification), `json` (metadata)
**Storage**: Files — Matrix Market (.mtx) format, JSON metadata, Git LFS for large matrices
**Testing**: Manual verification of matrix properties; automated metadata validation script
**Target Platform**: Linux (development container)
**Project Type**: Data collection (scripts + static assets, no application code)
**Performance Goals**: N/A (data preparation, not runtime)
**Constraints**: Total repository-hosted matrix data < 500MB (LFS); individual files > 50MB are download-on-demand only
**Scale/Scope**: 70+ matrices, 15+ hand-constructed, ~67 from SuiteSparse, size range 1x1 to ~1.6M x 1.6M

## Implementation Approach

### Phase A: Infrastructure Setup

1. **Install and configure Git LFS** (or document as prerequisite if container doesn't support it)
2. **Add LFS tracking rule** for `*.mtx` files to `.gitattributes`
3. **Create directory structure** under `sparse/test-data/`
4. **Initialize `metadata.json`** with empty schema

### Phase B: Hand-Constructed Matrices

Write `generate_hand_constructed.py` that produces:

1. **Arrow matrices** (5x5 PD, 10x10 indefinite, 15x15 indefinite):
   - Dense first row/column + diagonal entries
   - LDL^T factorization computed analytically via Schur complement

2. **Tridiagonal matrices** (5x5 PD, 10x10 indefinite, 20x20 indefinite):
   - Factorization via simple recurrence: d_i = a_i - b_{i-1}^2 / d_{i-1}

3. **Block diagonal** (15x15 with 3 blocks: 5+5+5):
   - Each block factored independently
   - One block PD, one indefinite, one with 2x2 pivots

4. **Bordered block diagonal** (20x20, adapted from SPRAL `gen_bordered_block_diag`):
   - Block structure with dense border requiring fill-in during factorization

5. **Stress-test matrices**:
   - Zero diagonal forcing 2x2 pivots (from SPRAL `simple_mat_zero_diag`)
   - Dense column causing worst-case fill-in
   - Ill-conditioned: near-singular with known exact factorization

6. **Degenerate cases**:
   - 1x1 trivial, 2x2 with 2x2 pivot
   - 3x3 singular (from SPRAL `simple_sing_mat`)
   - 3x3 null-column (from SPRAL `simple_sing_mat2`)

Each matrix gets a `.mtx` file and a `.json` file with the exact factorization. All factorizations verified numerically: check that `P^T A P - L D L^T` has norm < epsilon.

### Phase C: SuiteSparse Download Script

Write `download_suitesparse.py` with:

1. **Curated matrix lists** — three hardcoded lists based on APTP paper benchmark tables:
   - `EASY_INDEFINITE`: ~30 matrices from Duff et al. (2020) Table 1 + Hogg et al. (2016)
   - `HARD_INDEFINITE`: ~18 matrices from Duff et al. (2020) Table 2 + APTP-sensitive cases
   - `POSITIVE_DEFINITE`: ~19 matrices from Hogg et al. (2016) dagger-marked entries

2. **Download logic**:
   - Use `ssgetpy.search()` to find each matrix by group/name
   - Download in Matrix Market format with `format='MM'`
   - Extract to category subdirectories
   - Skip matrices > 50MB (mark as download-on-demand in metadata)

3. **CLI options**:
   - `--category`: Download only one category
   - `--max-rows`: Size filter
   - `--name`: Download specific matrix
   - `--verify-only`: Check existing files against curated lists
   - `--dry-run`: List what would be downloaded

4. **Metadata generation**: For each downloaded matrix, record SuiteSparse properties (id, group, kind, psym, nsym) and paper references.

### Phase D: Metadata Index Assembly

Write `validate_collection.py` that:

1. Scans all `.mtx` files in `test-data/`
2. Verifies each file is valid Matrix Market format
3. Checks metadata.json entries match actual files
4. Reports missing files, missing metadata, property mismatches
5. Generates summary statistics (count by category, size distribution)

Assemble the final `metadata.json` by merging:
- Hand-constructed metadata (from generation script)
- SuiteSparse metadata (from download script)
- Paper references (from research.md curated lists)
- Difficulty classifications and killer-case flags

### Phase E: Validation and Documentation

1. Run validation script to verify collection integrity
2. Verify SC-001 through SC-007 from spec
3. Update ssids-plan.md Phase 0.2 checkboxes
4. Update ssids-log.md with development notes
